# 현재 문제점 정리

---

## [수정 완료] 구조적 문제 (이번 세션에서 해결)

| 문제 | 파일 | 조치 |
|------|------|------|
| 모든 피처 OFF 프로세스가 supervisor에 미등록 → `set` 명령 NotFound | `main.cpp` | `registerProfile()`로 항상 등록 |
| disabled feature가 argv에 안 넘어감 → 런타임 ON 시 앱이 SHM 미연결 | `ProcessSupervisor.cpp` | 모든 feature ID 무조건 전달 |
| `setFeatureFlag`에서 꺼진 프로세스 자동 기동 없음 | `ProcessSupervisorFeature.cpp` | `isAliveLocked` 체크 후 `launchLocked` |
| 모든 피처 OFF 시 앱이 루프를 계속 도는 문제 | `SdvAppFrame.cpp` | `any_enabled` 체크 후 자율 종료 |
| 자율 종료 후 Watchdog이 재시작 반복 | `Watchdog.cpp` | `hasAnyEnabledFeature()` 체크 추가 |
| BUG-1: 재기동 후 피처 OFF 시 프로세스 미종료 (SHM unlink 타이밍) | `ProcessSupervisor.cpp` | SHM을 `registerProfile`에서 생성, 프로세스 수명과 분리 |
| DEBT-6: profile flag 3중 저장 | `ProcessRecord.h`, `Watchdog.cpp` | `ProcessRecord`에서 profile 제거, Watchdog이 `all_profiles_` 직접 참조 |
| DEBT-3: 소멸자 + Watchdog `waitpid` 경쟁 | `Watchdog.h/cpp`, `ProcessSupervisor.cpp` | `Watchdog::stop()` 추가, 소멸자에서 먼저 호출 |
| DEBT-2: `setKilled()` 미연결, `gracefulKill` is_killed 미설정, `hardKill` 타임아웃 1s | `ProcessSupervisorKill.cpp` | `gracefulKill`에 is_killed+SIGTERM 동시 설정, `hardKill` 200ms 단축, 재기동 시 is_killed 초기화 |
| 좀비 SHM 잔존 (비정상 종료 후 `/dev/shm/platform_*` 남음) | `ShmManager.cpp` | 생성자 첫 줄 `shm_unlink`로 선행 정리 |
| `--loop-ms` 파싱 예외 미처리 | `SdvAppFrame.cpp` | `try-catch`로 감싸기 |
| DEBT-1: `action_on_failure` 미사용 | `Watchdog.cpp` | `"DISABLE_FLAG"` / `"RESTART"` 분기 구현 |
| DEBT-4: `openForRead` 이름 불일치 (실제는 RDWR) | `ShmManager.h/cpp`, `SdvAppFrame.cpp` | `connect`으로 리네임 |
| DEBT-5: `ProcessRecord` 기본 state = Running | `ProcessRecord.h` | `Disabled`로 수정 |
