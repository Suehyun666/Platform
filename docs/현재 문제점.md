## [미해결] 설계 부채

### DEBT-1: `action_on_failure` 미사용
- **위치**: `FeatureProfile.h:8`, `ManifestLoader.cpp`, `Watchdog.cpp`
- **내용**: manifest에서 파싱하고 구조체에 저장되지만, Watchdog에서 실제 동작에 반영하지 않음. 항상 `Disabled`로 고정 처리.
- **영향**: 낮음 (현재 action이 하나라 문제없음)

### DEBT-2: `ShmManager::setKilled()` 미사용
- **위치**: `ShmManager.cpp:43`
- **내용**: 메서드는 있지만 supervisor 코드 어디서도 호출 안 함. `hardKill`이 SHM `is_killed`를 세팅해서 앱을 우아하게 종료시키는 설계였던 것으로 추정. 현재는 연결이 끊어진 상태.
- **영향**: 중간 (hardKill이 SIGKILL만 쓰게 되어 앱 cleanup 불가)

### DEBT-3: 소멸자 + Watchdog의 `waitpid` 경쟁
- **위치**: `ProcessSupervisor.cpp:19`, `Watchdog.cpp:32`
- **내용**: `~ProcessSupervisor()` 본체 실행 중 Watchdog 스레드가 아직 살아있음. 앱이 SIGTERM 받고 종료될 때 둘 중 하나만 `waitpid`로 수거 가능. Watchdog이 먼저 수거하면 소멸자의 `waitpid`는 `ECHILD` 반환 (현재 에러 체크 없음).
- **영향**: 높음 (간헐적 좀비 프로세스 또는 소멸 지연)

### DEBT-4: `ShmManager::openForRead` 이름 불일치
- **위치**: `ShmManager.cpp:49`
- **내용**: `O_RDWR`로 열고 있어서 앱에서도 SHM에 쓸 수 있는 상태. 이름이 Read 전용임을 암시해 오해 유발.
- **영향**: 낮음

### DEBT-5: `ProcessRecord` 기본 state = Running
- **위치**: `ProcessRecord.h:12`
- **내용**: 생성 직후 아직 실행 안 됐는데 기본값이 `Running`. 현재는 `launchLocked`에서 즉시 덮어쓰므로 문제없지만, 나중에 record를 미리 생성해두는 패턴이 생기면 Watchdog이 잘못된 판단을 내릴 수 있음.
- **영향**: 낮음
